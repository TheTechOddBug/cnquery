# ADR: cnspec SBOM Cataloger for Terraform Providers

**Status:** Proposed
**Date:** 2026-04-17

---

## Context

cnspec needs native SBOM cataloging for Terraform provider dependencies to detect which providers (and versions) are used in infrastructure-as-code configurations. Terraform providers are a supply chain vector — knowing which providers are pinned and at what versions is important for security auditing.

Terraform uses `.terraform.lock.hcl` to record resolved provider versions and integrity hashes. This file is generated by `terraform init` and committed to version control.

---

## File Formats

### .terraform.lock.hcl

```hcl
# This file is maintained automatically by "terraform init".

provider "registry.terraform.io/hashicorp/aws" {
  version     = "5.31.0"
  constraints = "~> 5.0"
  hashes = [
    "h1:ltYPMnfzETy+S3VDhirIBgt2sfY/gJFiCPWnTsSkN+k=",
    "zh:0cd0b11e5cafd84e05be4088ccf28a01a3dbae8e1ce71e3b91f7b27aca633b7b",
  ]
}

provider "registry.terraform.io/hashicorp/random" {
  version     = "3.6.0"
  constraints = ">= 3.0.0"
  hashes = [
    "h1:abc123=",
  ]
}
```

**Key fields per `provider` block:**
- Provider source address: `registry.terraform.io/<namespace>/<type>` (in the block label)
- `version`: Resolved version
- `constraints`: Version constraint expression
- `hashes`: Integrity hashes (h1: and zh: prefixes)

---

## Parsing Strategy

1. Read file line by line via `afero.Fs`
2. Detect `provider "source"` blocks by regex
3. Extract version from `version = "..."` lines within each block
4. Track block nesting via `{` and `}` counting
5. Skip comments (lines starting with `#` or `//`)

### Error Handling
- Malformed blocks are skipped with a debug log
- Missing version fields produce an entry with empty version
- Nested blocks (like `hashes`) are handled by brace counting

---

## Package Identification

### PURL
Format: `pkg:terraform/<namespace>/<type>@<version>`

Examples:
- `pkg:terraform/hashicorp/aws@5.31.0`
- `pkg:terraform/hashicorp/random@3.6.0`

The registry hostname (`registry.terraform.io`) is stripped from the source address since it's the default.

### CPE Generation
Vendor is the namespace (e.g., `hashicorp`), product is the provider type (e.g., `aws`).

---

## Dev/Prod Classification

Not applicable — all providers are production infrastructure dependencies.

---

## Remote Scanning Considerations

Lock files are small text (< 50KB typically). No special handling needed. All reads through `afero.Fs`.

Default search path: project root directory (`.terraform.lock.hcl`)

- Edge cases: custom registries, providers without version constraints
